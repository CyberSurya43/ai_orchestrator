# App Brief

Build a task manager for a small product team.

## Product Goal

Users need to create tasks, assign owners, set priority, track status, and quickly scan what needs attention.

## Core Workflows

- Create, edit, complete, and delete tasks.
- Filter tasks by owner, priority, status, and due date.
- View a compact dashboard of overdue, blocked, and high-priority work.

## Data

- Task: title, description, owner, priority, status, due date, created date, updated date.
- User: name and email.

## Constraints

- Preferred frontend stack: React + TypeScript.
- Preferred backend stack: FastAPI.
- Preferred database: SQLite for local development.
- Deployment target: Docker.

## Definition of Done

- The app runs locally.
- Core workflows are implemented.
- API tests cover task CRUD and filters.
- Deployment notes and Docker config are present.
